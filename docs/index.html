üí°Why Black Light AI?

Traditional vulnerability assessment has a signal-to-noise problem. Running tools like Nmap, Nuclei, and Amass often leaves you with a huge, fragmented JSON log that still needs hours of manual triage before you can tell what actually matters.

Black Light AI was built to fix three recurring bottlenecks:

  • Information overload ‚Äì Great tools, too much output. Thousands of findings with no clear prioritization.
  • Missing business context ‚Äì A scanner sees open ports; a human sees revenue, data and reputation at risk.
  • Reporting fatigue ‚Äì Turning raw tool output into an executive-friendly BLUF report is slow, repetitive work.

Black Light AI acts as a virtual security analyst: it ingests noisy scan data, reasons about risk, and produces decision-grade reports for both engineers and executives.

üß±Architecture Overview

Black Light AI implements a multi-agent pipeline rather than a single, monolithic LLM prompt. This keeps each step focused, reduces context size, and improves accuracy when working with large scan logs.

üöÄGetting Started

Quick Start

Clone the repository and open the Jupyter notebook to begin:

git clone https://github.com/ItsYourAnirban/Black-Light-Ai.git
cd Black-Light-Ai
jupyter notebook BlackLightAI.ipynb

Prerequisites

  • Python 3.8+
  • Jupyter Notebook or JupyterLab
  • Google API Key (for Gemini access)
  • System tools: Nmap, Go, Nuclei, Amass

Configuration

The main notebook includes configuration cells that allow you to:

  • Set RUN_LIVE_SCAN to enable/disable live scanning
  • Define TARGET_DOMAIN for reconnaissance
  • Enable SAFE_MODE for offline testing
  • Configure output directories and rate limits
